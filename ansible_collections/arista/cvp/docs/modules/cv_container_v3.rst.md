# cv_container_v3

Manage Provisioning topology.

Module added in version 3.0.0

<div class="contents" local="" depth="2">

</div>

## Synopsis

CloudVision Portal Configlet configuration requires a dictionary of
containers with their parent, to create and delete containers on CVP
side. Module also supports to configure configlets at container level.
Returns number of created and/or deleted containers

## Examples:

    # task in loose mode (default)
    - name: Create container topology on CVP
      hosts: cvp
      connection: local
      gather_facts: no
      vars:
        verbose: False
        containers:
            Fabric:
                parentContainerName: Tenant
            Spines:
                parentContainerName: Fabric
                configlets:
                    - container_configlet
                imageBundle: EOS-4.25.4M
      tasks:
        - name: 'running cv_container'
          arista.cvp.cv_container_v3:
            topology: "{{CVP_CONTAINERS}}"

    # task in strict mode
    - name: Create container topology on CVP
      hosts: cvp
      connection: local
      gather_facts: no
      vars:
        verbose: False
        containers:
            Fabric:
                parentContainerName: Tenant
            Spines:
                parentContainerName: Fabric
                configlets:
                    - container_configlet
      tasks:
        - name: 'running cv_container'
          arista.cvp.cv_container_v3:
            topology: "{{CVP_CONTAINERS}}"
            apply_mode: strict
